## Phase 1: 자동 업데이트 시스템 안정화 (Version & Download)
버전 비교 오류를 수정하고, 다운로드 실패 시의 사용자 경험을 개선한다.

- **Task 1-1 (버전 문자열 비교 오류 수정)** ✅:
  - `login.py`에서 로컬 버전과 구글 시트 버전을 비교할 때, 양쪽 문자열에 `.strip().lower()`를 적용하여 보이지 않는 공백이나 대소문자 차이로 인해 '다른 버전'으로 인식되는 버그를 해결한다. 버전이 완벽히 같으면 팝업 없이 즉시 메인 화면으로 진입하도록 보장한다.
- **Task 1-2 (수동 다운로드 Fallback 로직)** ✅:
  - 자동 다운로드 중 에러(네트워크 문제, 권한 문제 등)가 발생할 경우, 앱이 멈추지 않고 팝업을 띄우도록 `except` 블록을 강화한다.
  - 팝업에는 "자동 다운로드에 실패했습니다. 수동으로 다운로드해 주세요."라는 메시지와 함께 구글 시트에서 가져온 다운로드 링크를 텍스트로 보여주고, `webbrowser.open()`을 이용해 사용자의 기본 브라우저에서 해당 링크가 즉시 열리도록 구현한다.

## Phase 2: 데이터 동기화 및 중복 필터링 버그 픽스
UI에서 수정한 내용이 엔진에 즉시 반영되도록 하고, 고장 난 로컬 중복 필터링을 고친다.

- **Task 2-1 (수신처 삭제 UI-Data 실시간 동기화)** ✅:
  - 현재 Treeview(화면)에서 수신처를 삭제해도, 실제 발송 엔진으로 넘어가는 백그라운드 리스트(List/Dict)에는 삭제가 반영되지 않는 버그가 있다.
  - **[선택 행 삭제]** 버튼 클릭 시 Treeview에서 삭제함과 동시에, 엔진 구동에 사용되는 실제 데이터 변수(`self.recipients` 등)에서도 해당 인덱스/데이터를 완벽하게 `pop` 하거나 재생성하도록 로직을 수정한다.
- **Task 2-2 (템플릿 기반 중복 발송 로직 디버깅)** ✅:
  - A이메일에 A템플릿 전송 시 중복으로 걸러지지 않는 문제를 해결한다.
  - 발송 성공 시 `sent_log` DB에 `template_name`이 정확한 String 값으로 `INSERT` 되고 있는지 확인하고 교정한다. (빈 값이나 None으로 들어가는지 체크).
  - 발송 전 검사 쿼리(`SELECT 1 FROM sent_log WHERE email=? AND template_name=?`)에 바인딩되는 변수가 정확한지 철저히 점검하여, 동일 템플릿 재발송 시 무조건 "이전 발송 완료" 팝업/로그와 함께 스킵되도록 수정한다.

## Phase 3: 구글 시트 기반 글로벌 발송 이력 동기화 (Cloud Sent Log)
모든 직원이 발송 내역을 구글 시트로 실시간 공유하며, '담당자 영업권 보호'를 위한 2중 중복 필터링을 구현한다.

- **Task 3-1 (글로벌 발송 로그 시트 연동 및 스키마 업데이트)** ✅:
  - 구글 시트에 `worksheet("발송내역")` 탭을 추가하고, 헤더를 **[보낸 날짜, 발송담당자, 업체명, 이메일, 템플릿명]** 5가지로 구성한다.
  - 앱 실행 시 "발송내역" 시트를 읽어와 로컬 SQLite `sent_log` 테이블을 동기화한다. 이때 로컬 DB 스키마에도 반드시 `sender`(발송담당자) 컬럼이 존재해야 한다.
- **Task 3-2 (담당자 교차 발송 차단 2중 필터링 로직 구현)** ✅:
  - 발송 전 중복 검사 함수(`has_been_sent` 등)의 쿼리를 `SELECT sender, template_name FROM sent_log WHERE email=?` 로 변경하여 해당 이메일의 모든 발송 이력을 가져온다.
  - **[검사 규칙 1]**: 조회된 이력 중 `sender`가 현재 로그인한 담당자와 **다른 경우** ➡️ 즉시 스킵 처리하고, 로그에 `🚫 스킵: {업체명} (중복 발송입니다 - 타 담당자)` 출력.
  - **[검사 규칙 2]**: `sender`가 현재 로그인한 담당자와 **같지만**, `template_name`이 현재 보내려는 템플릿과 **같은 경우** ➡️ 즉시 스킵 처리하고, 로그에 `⏭️ 스킵: {업체명} (이미 동일 템플릿 발송됨)` 출력.
  - 위 두 조건에 해당하지 않을 때만 발송을 허용한다.
- **Task 3-3 (발송 완료 시 클라우드 실시간 기록)** ✅:
  - 발송이 '성공'할 때마다 구글 시트 "발송내역" 최하단에 데이터를 `append_row` 한다.
  - 데이터: `[현재시간, 현재담당자이름, 업체명, 이메일, 템플릿명]`

  ## Phase 4: 글로벌 데이터 동기화 및 시스템 안정화
로컬의 한계를 넘어 전 직원이 데이터를 공유하고, v2.5.1에서 발견된 잔버그들을 완전히 소탕한다.

- **Task 4-1 (버전 및 업데이트 로직 픽스)** ✅:
  - 버전 비교 시 `.strip().lower()`를 적용해 공백 오류를 차단한다. 버전이 같으면 팝업 없이 즉시 진입한다.
  - 다운로드 실패 시 에러만 띄우지 말고, 구글 시트의 **수동 다운로드 링크**를 보여주며 브라우저를 열어주는 Fallback 로직을 구현한다.
- **Task 4-2 (수신처 삭제 데이터 동기화)** ✅:
  - UI(Treeview)에서 행을 삭제했을 때, 엔진이 참조하는 실제 데이터 리스트(`self.recipients`)에서도 해당 항목이 즉시 제거되도록 로직을 일치시킨다.
- **Task 4-3 (글로벌 발송 내역 공유 및 2중 차단)** ✅:
  - 구글 시트 `worksheet("발송내역")`에 **[보낸 날짜, 발송담당자, 업체명, 이메일, 템플릿명]** 5개 컬럼으로 기록한다.
  - **[2중 필터링]**: 
    1) 타 담당자가 보낸 이력이 있다면 ➡️ "중복 발송입니다" (차단)
    2) 내가 보냈는데 템플릿까지 같다면 ➡️ "이미 동일 템플릿 발송됨" (차단)
    3) 내가 보냈지만 템플릿이 다르면 ➡️ 정상 발송 (허용)
- **Task 4-4 (스마트 중복 필터링 디버깅)** ✅:
  - 로컬 DB 기록 시 `template_name`이 누락되지 않는지 재점검하고, 중복 시 로그에 정확한 사유를 띄우도록 가독성을 높인다.

## Phase 5: 전문 배포 패키징 및 깃허브 자동화
사용자가 경고 없이 설치할 수 있게 패키징하고, 깃허브를 통해 업데이트 파일을 자동 공급한다.

- **Task 5-1 (Inno Setup 기반 설치 파일 제작)** ✅:
  - `.exe` 단일 파일 대신 정식 'Setup.exe' 설치 파일을 생성하는 스크립트를 작성한다.
- **Task 5-2 (GitHub Actions 배포 파이프라인 구축)** ✅:
  - 깃허브에 코드가 푸시되면 자동으로 Windows 환경에서 빌드(PyInstaller)를 수행하고, 생성된 파일을 Release 자산으로 등록하는 워크플로우를 생성한다.
- **Task 5-3 (업데이트 링크 동기화)** ✅:
  - 깃허브에 새 배포판이 올라가면, 해당 파일의 '직접 다운로드 주소'를 구글 시트의 `설정` 탭에 자동으로 업데이트하거나, 프로그램이 깃허브 API를 통해 최신 링크를 스스로 찾아오게 로직을 고도화한다.
